## 重点项目简述

### 1）基础数据服务(2025-至今)
```
【项目简介】：负责微信基础数据合规体系建设，实现海量数据的加密存储与高效回溯，支撑BG内外的安全运营需求。
【相关数据】：日均处理80+亿条，单场景峰值1000w/min；服务集群节点约120个(1000核)；ES存储约400TB；
【项目难点】：
- 1）数据写入：支持多场景、多链路、异构数据的准实时写入，链路数据不丢、故障可恢复；
- 2）密文搜索：实现链路的密文传输、明文检索，满足合规要求（业界较少可参考的案例）；
- 3）回溯查询：支持对文本和媒体数据的相似查询(召回率要求>98%)，满足业务的溯源扩散类指令；
【解决方案】：
- 1）数据写入：构建mq+mmdata+es+flink的多路数据处理架构；实现mmdata代理支持旧链路的数据上报；
- 2）密文搜索：改造es的分词插件、实现密文保存+明文索引；
- 3）回溯查询：集成Astra中台DHN特征提取+FAISS算法实现毫秒级媒体检索；基于es实现文本回溯接口；
```


### 2）关键词服务(2024-至今)
```
【项目简介】：构建亿级关键词实时过滤系统，包含：文本过滤服务、词库运营服务、种子互通等模块。
【相关数据】：单机峰值100+w/min、集群节点200+个(约1.2w+核)；词库量级6000+w、单词库最大300+w；
【项目难点】：
- 1）数据互通：外部门异构词库数据的写入；业务高效接入；复杂度可维护；
- 2）运营优化：大词库的容量管理；节点的腾挪和调度；
- 3）生效优化：缩短种子从添加到服务生效的耗时；确保每一个种子都不会漏过或误杀；
【解决方案】：
- 1）数据互通：类网络协议的配置化设计和分层架构，支持字段枚举映射与函数逻辑取值；业务接入效率提升10倍；
- 2）运营优化：词库拆分；冷热分离；文本降噪/预处理（转换+过滤）；类k8s-hpa的弹性调度机制；
- 3）生效优化：全量+增量双加载模式；基于delaycheck旁路服务实现种子的异步生效验证和告警；
```

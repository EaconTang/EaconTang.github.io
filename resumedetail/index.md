## 重点项目简述

### 1）基础数据服务(2025-至今)
```
【项目简介】：负责微信基础海量数据的合规存储、加密、和回溯查询，支撑BG内外的安全运营需求
【相关数据】：全场景约80+亿/天、单场景峰值约1000w/min；服务集群节点约120个(1000核)；ES存储约400TB；
【项目难点】：
- 1）数据写入：支持多场景、多链路、异构数据的准实时写入，链路数据不丢、故障可恢复；
- 2）密文搜索：实现链路的密文传输、明文检索，满足合规要求（业界较少可参考的案例）；
- 3）回溯查询：支持对文本和媒体数据的相似查询(召回率要求>98%)，满足业务的溯源扩散类指令；
【解决方案】：
- 1）数据写入：基于mq+mmdata+es+flink实现多路数据的统一处理架构；实现mmdata代理支持旧链路的数据上报；
- 2）密文搜索：改造es的分词插件实现密文保存+明文索引；提供单独的解密服务；
- 3）回溯查询：基于astra中台的DHN特征提取、FAISS特征检索算法实现媒体数据的相似查询；基于es实现文本回溯接口；
```


### 2）关键词服务(2024-至今)
```
【项目简介】：维护关键词数据服务、对海量请求进行匹配和拦截，包括：关键词过滤服务、词库运营、种子互通等模块。
【相关数据】：单机峰值100+w/min、集群节点200+个(约1.2w+核)；词库量级6000+w、单词库最大300+w
【项目难点】：
- 1）数据互通：外部门异构词库数据的写入；业务高效接入；复杂度可维护；
- 2）运营优化：大词库的容量管理；节点的腾挪和调度；
- 3）生效优化：缩短种子从添加到服务生效的耗时；确保每一个种子都不会漏过或误杀；
【解决方案】：
- 1）数据互通：类网络协议的配置化和分层设计、支持字段和枚举值的映射；支持变量/常量/函数逻辑的取值模式；
- 2）运营优化：词库拆分；冷热分离；文本降噪/预处理（转换+过滤）；类k8s-hpa的弹性调度机制；
- 3）生效优化：定期全量+实时增量加载；基于旁路delaycheck服务实现种子生效的一致性告警；
```
